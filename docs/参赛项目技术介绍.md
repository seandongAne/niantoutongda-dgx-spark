# AI 搬家复原 Agent：把一个家的秩序，编译进另一个家

我们把搬家理解为一次生活秩序的迁移：机器需要看懂旧家里有什么，听懂哪些物品属于同一种生活习惯，再走进新家，给出可以直接执行的复原方案。

围绕这个目标，我们在一台 NVIDIA DGX Spark 上实现了完整的多模态搬家复原系统。它接收三段旧居视频、一段口述和一段新居扫描，最终输出可信物品记忆、搬运组合、新居布局和 5 张可执行任务卡。输入是视频与自然语言，输出是一套可以落到现实空间里的行动计划。

## 把视频变成“家庭世界模型”

Step-Audio 2 mini 首先将口述整理成结构化生活线索；Grounding DINO 在视频中完成开放词汇目标发现，DINOv2 冻结视觉骨干配合轻量投影头提取跨视角特征，再通过匈牙利匹配、质量门和 ReID 约束，把同一件物品散落在不同镜头里的视觉碎片重新连接起来。

这套感知链建立的是一份带身份、关系、置信度和原始证据的物品记忆。系统既知道“看见了什么”，也知道“哪些东西应该一起搬”。旧居视频由此被转换成一个可计算的家庭世界模型，为后续空间推理提供统一事实底座。

## 四个 Agent，把记忆编译成行动

主链由四个职责明确的 Agent 协同完成。MEM Agent 维护物品身份、跨镜追踪和最小澄清问题；GROUP Agent 从口述与视觉证据中恢复生活组合；SPACE Agent 将新居视频转换为候选区域图，并调用 OR-Tools CP-SAT，在容量、支撑、电源、共置和互斥等约束下求解布局；EXEC Agent 把最终方案编译成任务卡，并调度验收。

四个 Agent 都拥有明确的数据合同和进程边界。Pydantic v2 定义 JSON/JSONL 消息结构，每条消息携带 `message_id`、因果引用、生产者与 SHA-256 载荷哈希。一张任务卡可以一路回溯到空间候选、物品实体和原始视频帧。验收阶段，MEM 与 SPACE 已在 Spark 上以独立进程并行 fan-out，再由 EXEC 完成确定性 fan-in；整条因果链支持严格回放。

## 一台 Spark，就是完整的 AI 工作站

DGX Spark 的 GB10 与 128 GB 统一内存承载了语音理解、开放词汇检测、视觉嵌入、视觉语言模型、组合优化和 Agent 编排。Nemotron Nano 12B V2 VL 采用 NVFP4 权重并通过 vLLM 服务化，图像任务吞吐从约 1.7 tok/s 提升到 25.4 tok/s，达到原始 BF16 Transformers 路径的约 15 倍。

调优阶段，StepFun 3.7 Flash 承担难样本归因和伪标签建议，Spark 本地完成 DINOv2 投影头训练与工作点搜索。本轮自动处理 446 个困难或不确定样本对，构造 117 个伪身份、511 个训练样本，并扫描 18 组工作点，将需要人工澄清的问题从 1379 压缩到 677，减少约 51%。

TensorRT 视觉推理部署链路正在测试，覆盖引擎构建、输出一致性和吞吐表现；测试结果将直接进入同一套可回放证据链。

## 3306 → 20 → 5：从视频到现实动作

在完整实测中，系统从 **3306 个原始视觉实体中建立了 20 件可信物品记忆**，把确认问题控制在 4 个以内；随后形成 **3 个生活组合与 2 个独立装箱单元**，完整覆盖 20/20 件物品。新居侧从 **2278 条空间观测中生成 168 个候选区域**，自动确定 5 个最终放置区域，独立空间评分达到 5/5。

约束求解输出 `PLAN_READY`，5 个搬运单元被编译为 5 张任务卡；四 Agent 主链严格回放结果为 `PASS`，交付包包含 44 份经过 SHA-256 校验的阶段产物。至此，旧居里的声音、物品和关系，被完整编译成新居中可执行、可验收、可追溯的一组动作。

这是一套能理解生活关系、推理真实空间并驱动现实任务的本地 AI 系统。一台 Spark，完成了从多模态感知到物理世界行动计划的整条闭环。
