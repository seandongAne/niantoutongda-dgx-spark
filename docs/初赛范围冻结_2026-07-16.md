## 1. 初赛唯一目标

在**一个授权、整洁、受控的卧室搬家任务**中真实跑通并清楚展示：

1. 三段旧家引导视频形成物品观测与单视频轨迹；
2. 同一件物品跨视频聚合为稳定实体；
3. 睡前组合、学习组合、收纳或充电组合关系清晰形成；
4. 新家候选区域、确定性布局、任务卡与验收 trace 完整闭环；
5. 成果可复跑，演示视频与十日谈引用真实输入、配置、日志、metrics 和截图。

这是一条**受控场景的任务级视觉记忆与生活关系复原**主张，不是通用家居识别主张。

## 2. 受控英雄场景合同

- 只保留一个初赛英雄任务；开发素材可继续作为诊断证据，但不再扩展成多房间矩阵。
- 12～15 件锚点，轮廓清楚、尺寸适中、互不遮挡、不堆叠。
- 每件锚点至少出现在三段视频中的两段；拍摄时完整入画并短暂停留。
- 三个生活组合边界清楚，每组 3～4 件物品，组合间留出空间。
- 可保留 1～2 组同类但外观可区分的物品，证明实例级聚合；不主动制造极端困难场景。
- 光线稳定、慢速平移、先远景再近景；演示允许剪辑，但画面中的系统结果必须来自真实管线产物。

## 3. 立即停止

| 停止项 | 初赛处理 |
|---|---|
| 5000 张网络或生成图片 | 不采集、不生成、不标注、不建立大规模数据工厂 |
| Grounding DINO 全参数微调 | 不搭 TAO/ODVG 训练管道，不训练 GDINO 权重；继续使用冻结 checkpoint 与已验证推理配置 |
| 多房间通用性验证 | 不建立房型矩阵，不以跨房间指标作为完成门 |
| R1 大规模主动学习 | 不启动 R1 数据池、challenge、云端增强或循环训练 |
| 正式任务 B 盲测 | 取消初赛硬门；不建立封存答案、预注册后盲跑或据此宣称泛化 |
| “任意家庭、任意物品都能识别”的产品级证明 | 不开发、不测试、不写入成果、演示视频或十日谈主张 |

停止意味着不再投入新的拍摄、标注、云额度、Spark 训练时间或工程时间。已经存在的文档、
代码和结果保留作历史证据，不继续扩建，也不得反向成为初赛阻塞项。

## 4. 继续完成

- 冻结 GDINO 推理词表、分批、NMS、tile 与拍摄工作点；只修 P0 正确性缺陷。
- 完成 S3 低噪声跨视频聚合与必要的轻量投影头训练；不训练 GDINO 全模型。
- 完成生活组合、区域图、CP-SAT 布局、结构化任务卡与验收复核消息族。
- 完成可点击成果、3～4 分钟演示视频和十日谈；核心闭环优先压进 90 秒。
- 保留一次稳定复跑、0 高置信误合并、真实 trace 与前后对照，作为可信度底线。
- StepFun 只保留已进入主线的低成本用途；不新建网络图/生成图规模化校准任务。

## 5. 对外能力边界

允许表述：

> 系统通过一次引导式扫描，为一个具体搬家任务建立经过验证的物品视觉记忆，
> 并把清晰的生活组合关系转化为新家区域级执行计划。

禁止表述：

- “已经适配任意家庭或任意家居物品”；
- “任务 B 盲测通过”或“已证明跨房间泛化”；
- “用 5000 张网络/生成图片训练”或“已全量微调 GDINO”；
- 以手工修改实体 ID、组合或布局冒充系统自动结果。

## 6. 初赛完成门

- 一个受控英雄任务从视频输入到验收 trace 完整跑通；
- 演示视频使用真实成果，画面、旁白与技术主张一致；
- 十日谈按日记录真实证据、失败与取舍，并明确本次范围冻结；
- 所有未完成的泛化、盲测与大规模训练内容均写成“停止/未做”，不包装成未来时完成项。
