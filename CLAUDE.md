# DGX Spark 黑客松 — 工作流铁律

> NVIDIA DGX Spark 黑客松(2026-07-12 ~ 2026-07-22),队名:念头通达。
> 以下规则适用于本项目的所有会话,无例外。

## 架构

- **Claude Code 只在本地(这台 Mac)运行**;远程 spark(DGX Spark, GB10 / aarch64 / 128GB 统一内存)是唯一的 GPU 执行场。
- 所有 GPU / 推理 / 训练命令统一通过 `ssh spark 'cd ~/proj && ...'` 执行。
- SSH 已配置 `Host spark`(ControlMaster 复用连接,4h 保活);跨境网络不稳定是常态,架构必须对断连免疫。

## 文件纪律

- 代码**只在本地编辑**,绝不在远程直接改文件。
- 改动通过 `scripts/deploy.sh` 部署到 `spark:~/proj/`(rsync --delete,远程手改会被覆盖——这是特性不是缺陷)。

## 长任务纪律

- 预计 **>1 分钟**的远程任务一律后台发射:
  `ssh spark 'cd ~/proj && nohup <命令> > ~/proj/logs/<任务名>.log 2>&1 &'`
- 之后轮询进度:`ssh spark 'tail -n 50 ~/proj/logs/<任务名>.log'`
- **绝不挂前台**等待长任务;nohup + 日志天然免疫断连。

## 模型纪律

- 权重**只在 Spark 上从 ModelScope 拉取**。
- **绝不经 SSH/rsync 传大文件**(主办方明令禁止,影响他人使用)。
- 本地不存权重(.gitignore 已屏蔽 models/、*.safetensors 等)。

## 内存纪律

- 任何加载模型的操作**前**,先 `ssh spark 'free -h'` 并向用户汇报。
- OOM 是这台机器最常见的死因;主办方提示:网络正常但连不上时,90% 是内存溢出。

## 成果保险

- 每个有意义的改动立即本地 `git commit`。
- 远程小体积产出(adapter / 日志 / 评测结果)及时用 `scripts/pull_results.sh` 拉回。
- 把远程机器当作**随时可能被回收的临时执行器**;任何只存在于远程的东西都视为已丢失。

## 网络现实

- 跨境单条命令失败,**先重试一次**再下结论。
- 长任务靠 nohup + 日志,不依赖连接存活。
