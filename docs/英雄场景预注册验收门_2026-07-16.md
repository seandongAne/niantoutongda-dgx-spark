> 预注册的意义:**门在看到素材之前定死**。素材到手后只做判卷,不做挪门;
> 任何门槛调整都必须以新版本文档明文记录"改了什么、为什么、谁批准",
> 否则"验收通过"这句话在演示和十日谈里不许出现。
> 上位裁决:`docs/初赛范围冻结_2026-07-16.md`;拍摄合同:`docs/英雄房间拍摄要求.md`。

## 0. 判卷物料与口径

- 素材:旧房间 3 段 + 新家 1 段 + 逐件旁白(拍摄要求 §二)。
- GT 制作:沿用 dev_a 对账工具链(锚点点选式对账 UI)对英雄房间全部锚点
  确认 anchor→tracklet 映射;同款不同色对自动导出为硬负真值。
- 判卷程序:`anchor_gt_eval` 同族脚本 + `hero_pipeline` 产物;
  所有数字以判卷脚本输出为准,不接受目测口径。
- 预注册候选工作点(二选一,判卷定,不新调):
  - **W1 = v3 冻结口径**(attr 权重 0.00,m0.84/g0.00);
  - **W2 = attr 0.15**(dev_a 上 R@1 +2pp/澄清 −12/零误合并)。
  - 决策规则:两点都判卷,G2 全过者取 R@1 高的;都不过走 §4 校准预算。

G1 不过 = **素材问题,不进算法判卷**,按拍摄要求 §五自查清单退回重拍/补拍。

## 2. G2 聚合门(核心)

| 门 | 阈值 | 性质 |
|---|---|---|
| G2a 误合并 | 0(跨锚点自动链接零容忍) | 硬门 |
| G2b 硬负对 | 同款不同色对 0 互并(全部对) | 硬门 |
| G2c 完整合并 | ≥80% 锚点的全部确认轨合并为单实体(12~15 件即 ≥10/12…≥12/15) | 主门 |
| G2d R@1 | ≥0.90 | 主门 |
| G2e 澄清预算 | 澄清请求 ≤ 2×锚点数 | 主门 |

(轮廓清楚/不遮挡/不堆叠/停留 1-2s)就是为消解碎轨设计的,G2c/G2d 是对
"场景合同 + 冻结工作点"联合命题的检验,不是对模型泛化的检验。

## 4. 校准预算(预注册,防"调到过为止")

素材到手后允许的全部调整动作,限于以下白名单,**每一项动用都记档**:

1. W1/W2 二选一(§0 决策规则);
2. 词表编译:英雄房间 12~15 件锚点词按既有词表原则重编(分批 ≤4 易混词),
   走 48 候选同款打分回路,不引入新机制;
3. stitch 阈值在 {0.83, 0.85, 0.87} 三点内选一(dev_a 冻结 0.85 为缺省);
4. 澄清互选封顶 {2, 3} 二选一。

白名单之外(匹配阈值、margin、min_quality、权重自由调、改代码逻辑)一律
视为挪门,需要新版本文档 + 明文理由。**允许一次整体校准迭代**:第一轮判卷
不过 → 动用白名单 → 第二轮判卷为终局;两轮都不过,如实写入十日谈
("受控场景下 X 门未过"),演示按实际达成水平剪。

## 5. 表述红线(继承范围冻结 §5)

- 通过本验收门只支持表述:**"在受控英雄场景与预注册验收门下全链路达标"**;
- 禁止:任意家庭适配 / 盲测通过 / 5000 图训练 / 全量微调 / 手工冒充自动结果;
- 校准就叫校准,不叫训练;没跑 TAO 不写 TAO。
