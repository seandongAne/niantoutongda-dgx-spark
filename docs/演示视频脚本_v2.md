# 演示视频脚本 v2 · 把旧家的生活组合带到新家

> 目标时长：105–115 秒。
>
> 当前唯一正式入口：`results/hero/s1-auto-final-v1/index.html`。
>
> 叙事顺序：生活问题 → 可信库存 → 生活组合 → 自动空间 → 可执行任务 → 模型优化与审计。
>
> 红线：只展示已归档证据；StepFun 仅为开发期调优代理，演示运行时保持 Spark 本地、零云依赖。

## 一句话故事

搬家不只是把物品运过去，而是把熟悉的生活秩序重新拼起来。对旧家和新家各拍几段
视频，系统将嘈杂识别结果收敛为可信物品，理解哪些东西共同使用，再自动选择新家的
相对区域，生成搬家人员可以直接执行的任务卡。

## 录屏顺序

1. 从正式成果页首屏开始，不从终端或 JSON 开始。
2. 点击“30 秒看懂闭环”，依次录制四张故事卡。
3. 点击“新家五区”，录制三张真实证据帧和 `PASS / 5/5 / 0 待人工区域`。
4. 点击“代表任务”，只放大学习文具箱到书桌；随后用一次短镜头交代五卡覆盖 20/20。
5. NVIDIA 与 StepFun 证据各保留一屏，不滚完整日志或 sweep 表。
6. 收尾展示四 Agent 主链和 replay `PASS`，再回到标题页。

## 主片不展示

- 不滚动 46 行 SHA-256 表、完整 JSON、风险延期表和精确几何范围；这些保留在“完整证据”与答辩页。
- 不展示旧的 `s1-final` 或 `s1-technical-closure` 页面；它们分别代表旧库存口径和人工视觉阶段。
- 不再使用“681 条消息重放”或“全包三次 hash 一致”的旧说法。当前正式主链是四条因果消息闭合，replay 为 `PASS`。
- 不说“VLM 命名权威”“完全理解家庭习惯”“最佳且唯一布局”或“保证儿童与用电安全”。
- 不把 R@1 0.808 和 14/20 做成收尾失败数字卡。若评委追问通用 ReID，再在答辩页诚实说明该门未达；主演示证明的是可信投影后的完整英雄任务。
- 不把真实摆放照片当作当前主链必需证据；物理执行复核继续作为可选延期项。

## 答辩时的边界说法

> 当前已经完成并验证的是比赛级规划闭环：可信库存、生活组合、自动空间、约束布局、
> 任务卡和审计回放。它不等于任何家庭、任何房间都能无确认完成，也不构成物理摆放
> 或安全认证。
